Driver Module

GRAINSCO Fieldproxy delivery management application — manage delivery orders, drivers, and vehicles from one dashboard.

Overview

GRAINSCO Fieldproxy delivery management application

The Driver Module is a delivery management system built on the Fieldproxy platform, branded for GRAINSCO. The app is designed to manage delivery orders, drivers, and vehicles from a central dashboard.

Sidebar navigation

The left-side navigation has 4 items (collapsible):

  • All Deliveries — delivery box icon
  • Create Order — plus icon
  • Drivers — person icon
  • Vehicles — truck icon
Watch the Driver Module walkthrough

All Deliveries

Driver Module / /all_deliveries_view

The main dashboard where you can monitor and manage every delivery order in the system.

Top section — Summary cards (5 KPI tiles)

  • All Orders — total count (currently 27)
  • Pending — orders awaiting assignment/dispatch (currently 8)
  • In Transit — orders actively being delivered (currently 0)
  • Completed — successfully delivered orders (currently 16)
  • Failed — failed delivery attempts (currently 0)

Date filter

A date range picker (dd MM yyyy – dd MM yyyy) with an Apply button to filter the delivery list by date range.

Delivery orders table

Shows all orders with these columns:

  • Order # — unique ID (format: DLV-YYYY-XXXX, e.g. DLV-2026-0027)
  • Customer — the recipient/customer name
  • Address — delivery destination
  • Driver — assigned driver (with a dropdown to change)
  • Status — color-coded badge: Completed Pending In Transit Arrived Failed
  • Priority — color-coded: Urgent High Normal
  • Created — date the order was created
  • Actions — eye icon to open the detail view of each order

Additional features

  • Each column has a "Contains" filter row for searching/filtering within the column
  • Export button ( download icon) — allows exporting the table data
  • Pagination at the bottom (27 orders spread over 2 pages, with rows-per-page selector)
  • "+ New Delivery Order" button in the top right (takes you to the Create Order view)

Delivery Detail

Driver Module / /delivery_detail_view — sub-view opened from All Deliveries

Shows the complete information for a single delivery order.

How to access

Click the eye icon on any row in the All Deliveries table.

Top header

  • Breadcrumb navigation: ← Back to Deliveries / DLV-2026-XXXX
  • Order number as title (e.g., DLV-2026-0027)
  • Customer name and company subtitle
  • Status badge (e.g., Completed)
  • "Generate PDF" button (purple) — generates a PDF report for the order

Information panels (3 columns)

Panel 1 — Customer & Contact

  • Customer name
  • Company
  • Contact Person
  • Primary Mobile
  • Secondary Mobile
  • Email

Panel 2 — Delivery Address

  • Address
  • Nearest Landmark
  • City
  • Region

Panel 3 — Delivery Details

  • Time Window
  • Priority (Urgent / High / Normal)
  • Special Instructions

Second row of panels

Assignment Panel

  • Assigned Driver
  • Assigned Vehicle

Status Panel

  • Order Status
  • Cargo Verified (true / false)

Completion Panel

  • Trip Ticket Issued (true / false)
  • Filed Off (true / false)

Bottom tabs (5 tabs)

Tab 1 — Items

Shows all the physical goods in the delivery order in a table with:

  • Item name
  • Description
  • Quantity (QTY)
  • Weight
  • Unit (Boxes, Pcs, etc.)
  • Handling notes
  • "+ Add Items" button to add new items to the order

Tab 2 — Timeline

A chronological event log showing every step of the delivery lifecycle:

  • Delivery Signed Off & Closed
  • Keys Returned & Trip Completed
  • Post-Trip Inspection Completed
  • Return to Warehouse
  • Arrived at Site (with timestamp)
  • Manual Status Update
  • Departed to Destination
  • Trip Ticket Issued
  • Cargo Verified & Loaded
  • Pre-Trip Inspection Completed

Each event shows: event name, description, who performed it (By: [driver name]), and timestamp.

Tab 3 — Expenses

Lists all expenses associated with the delivery:

  • Date / Timestamp
  • Type (e.g., "Fuel Receipt")
  • Amount (e.g., 45688.0)
  • Description (e.g., "petrol")

Tab 4 — All Inspections

Shows vehicle inspection forms submitted for the delivery:

  • Pre-Trip Inspection with Driver name, Vehicle name, and Odometer reading
  • Inspection checklist by Category (e.g., "Exterior Check"), Check Item (e.g., "Vehicle body free from major dents/damages"), and Result (Yes / No)

Tab 5 — PDF

Renders a live PDF preview of the Delivery Report directly in the browser. The PDF includes:

  • Company logo and branding (GRAINSCO)
  • Header: Order No, Date, Driver, Vehicle, Status
  • Section A: Customer Information
  • Section B: Delivery Items table

Create Order

Driver Module / /create_delivery_order

A form to create a brand new delivery order from scratch.

Header

"Create New Delivery Order" with subtitle: "Step 1: Fill in customer and delivery details, then add items on the next page"

Section 1 — Customer Information

  • Customer Name* (required)
  • Company Name
  • Contact Person
  • Email
  • Primary Mobile* (with country code selector, default US +1)
  • Secondary Mobile (with country code selector)

Section 2 — Delivery Address

  • Delivery Address* (required)
  • Region / Province* (dropdown)
  • City / Municipality* (dropdown, depends on Region)
  • Zip Code
  • Nearest Landmark

Section 3 — Delivery Details & Assignment

  • Scheduled Delivery Date (date picker)
  • Delivery Time Window* (required, text field)
  • Priority* (dropdown: Urgent / High / Normal)
  • Assign Driver* (dropdown list of available drivers)
  • Assign Vehicle* (dropdown list of available vehicles)
  • Special Instructions (text area)

Footer buttons

  • Cancel — goes back to deliveries list
  • → Create & Add Items — saves the order and proceeds to Step 2 (adding delivery items)

Drivers

Driver Module / /drivers_view

Manage all drivers registered in the system and their profiles.

Top section — Summary cards (4 KPI tiles)

  • Total Drivers — total registered drivers (currently 1)
  • Active — currently active drivers (currently 1)
  • Inactive — deactivated drivers (currently 0)
  • Available Now — drivers available for assignment right now (currently 1)

All drivers table

Shows each driver with columns:

  • Driver Name
  • Phone
  • Email
  • License # (license plate / number)
  • License Type
  • Blood Group
  • Availability (Available / Unavailable)
  • Status — color-coded badge (Active)
  • Added — date added to the system
  • Actions — delete button

Add Driver button (top right)

Opens a modal popup — Add New Driver — with the following fields:

  • Driver Name
  • Phone (with country code selector)
  • Email
  • License Number
  • License Type
  • Blood Group
  • Emergency Contact Name
  • Emergency Contact Phone
  • Password (for driver's mobile app login)
  • Cancel / Save buttons

Vehicles

Driver Module / /vehicles_view

Manage the entire vehicle fleet, including motorcycles, pickups, and trucks.

Top section — Summary cards (4 KPI tiles)

  • Total Vehicles — total fleet count (currently 43)
  • Active — currently active vehicles (currently 43)
  • Inactive — decommissioned vehicles (currently 0)
  • Available Now — vehicles ready for assignment (currently 43)

All vehicles table

Shows each vehicle with columns:

  • Vehicle # (internal code, e.g. CDO-011, MC-001)
  • Plate No. (license plate)
  • Type (Pickup, Motorcycle, etc.)
  • Make (Toyota, Honda, Kawasaki, Isuzu, Rusi, etc.)
  • Model (Hilux, Wave 110, D-Max, etc.)
  • Year
  • Capacity (kg)
  • Fuel (fuel type)
  • Odometer
  • Availability
  • Status — badge (Active)
  • Added — date added
  • Actions — delete button

"+ Add Vehicle" button (top right)

Opens a modal popup — Add New Vehicle — with the following fields:

  • Vehicle Number (internal ID)
  • Plate Number
  • Vehicle Type (dropdown)
  • Make (manufacturer)
  • Model
  • Year
  • Capacity (kg)
  • Fuel Type (dropdown)
  • Insurance Number
  • Current Odometer
  • Cancel / Save Vehicle buttons

End-to-end summary

The system is designed to give dispatchers and managers a full end-to-end view — from creating an order, assigning it to a driver and vehicle, tracking it through its lifecycle, recording expenses and inspections, all the way to generating a final delivery PDF report.
